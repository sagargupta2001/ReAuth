# Memory Docs (ReAuth)

Purpose: compact, structured project memory for fast onboarding of humans and agents.

## How to use
- Start with `00-context.md`.
- Read `01-architecture.md` and `03-control-flow.md` for mental model.
- Use `04-oidc-sso-flows.md` for protocol flows.
- Use `09-roadmap.md` for next steps.

## Conventions
- Keep files concise. Prefer bullet lists and diagrams.
- When adding a decision, add an ADR under `adr/` and list it in `adr/README.md`.
- When naming paths, use repo-relative paths from the repository root.
